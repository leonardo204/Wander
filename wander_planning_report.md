# Wander 서비스 기획 리포트

## 1. 서비스 개요

### 1.1 서비스 정의
**Wander**는 사용자가 iPhone 사진앱에서 선택한 사진들의 메타데이터(GPS, 시간 등)를 AI가 분석하여 사용자의 동선, 활동, 상황을 자동으로 추측하고 스토리로 만들어주는 서비스입니다.

### 1.2 핵심 컨셉
- **"너 지금 뭐해?"** 에 대한 답변을 사진으로 하는 서비스
- 사진 몇 장만 올리면 AI가 여행인지, 일상인지, 모임인지 자동 파악
- 메타데이터 기반 분석 + AI 추론의 결합

### 1.3 타겟 사용자
| 구분 | 설명 |
|------|------|
| 1차 타겟 | 여행을 자주 다니며 기록을 남기고 싶은 2030 세대 |
| 2차 타겟 | 일상을 SNS에 공유하고 싶지만 글쓰기가 번거로운 사용자 |
| 3차 타겟 | 친구/가족에게 현재 상황을 쉽게 공유하고 싶은 사용자 |

## 2. 시장 조사 및 경쟁 서비스 분석

### 2.1 유사 서비스 분류

#### A. 여행 기록/추적 앱

| 서비스명 | 주요 특징 | 강점 | 약점 |
|----------|----------|------|------|
| **Polarsteps** | GPS 자동 추적, 여행 타임라인 생성, 포토북 제작 | 자동 경로 추적, 오프라인 지원, 배터리 효율 (일 4%) | 실시간 추적 필요, 과거 여행 입력 어려움 |
| **FindPenguins** | GPS 추적, AI 기반 자동 저널, 3D 플라이오버 영상 | AI 활용, 커뮤니티 기능, 무료 기본 기능 | 데이터 연결 필요 시 제한 |
| **Journi Blog** | 자동 사진 정리(시간/위치), 포토북 제작, 협업 기능 | 자동 그룹핑, 여행 공유, 고품질 출력 | 프리미엄 기능 유료 |
| **TripIt** | 예약 이메일 파싱, 일정 자동 생성 | 예약 정보 통합, 비즈니스 여행 최적화 | 사진/스토리텔링 약함 |

#### B. 실시간 상태 공유 앱

| 서비스명 | 주요 특징 | 강점 | 약점 |

#### C. 사진 메타데이터 도구

| 서비스명 | 주요 특징 |
|----------|----------|
| **EXIF Metadata (iOS)** | GPS, 시간, 카메라 정보 조회/편집 |
| **Photo Metadata Viewer** | 다양한 메타데이터 태그 확인 |

## 3. Wander 차별화 포인트

### 3.1 기존 서비스의 Gap

| 기존 서비스 한계 | Wander 해결 방안 |
|------------------|------------------|
| 실시간 GPS 추적 필요 → 배터리 소모, 사전 설정 필요 | 사후 사진 선택만으로 동선 재구성 |
| 사진만 정리, 맥락 파악은 사용자 몫 | AI가 "무엇을 했는지" 자동 추론 |
| 여행 전용 or 일상 전용으로 분리 | 여행/일상 모두 커버하는 통합 서비스 |
| 글쓰기 필요 (캡션, 저널) | 메타데이터만으로 스토리 자동 생성 |

### 3.2 Wander의 핵심 차별점

```
┌─────────────────────────────────────────────────────────────────┐
│                     🚀 Wander 핵심 차별점                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│  1. Zero-Setup 여행 기록                                          │
│     • GPS 추적 앱 켜둘 필요 없음                                   │
│     • 여행 후에도 사진만 선택하면 자동 분석                         │
│                                                                   │
│  2. AI 맥락 추론                                                  │
│     • 위치 + 시간 + 장소 유형 조합으로 활동 추측                    │
│     • "카페에서 브런치" → "명동에서 쇼핑" → "한강에서 피크닉"        │
│                                                                   │
│  3. "지금 뭐해?" 실시간 답변 모드                                  │
│     • 사진 몇 장 → 즉시 상황 요약 생성                              │
│     • SNS 공유 최적화 포맷                                         │
│                                                                   │
│  4. 여행 + 일상 통합                                               │
│     • 여행: 전체 여정 타임라인                                      │
│     • 일상: 오늘 하루 요약, 주간 하이라이트                         │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

## 4. 기능 정의 (Feature Specification)

|------|------|----------|
| **사진 선택 & 메타데이터 추출** | iOS 사진앱 연동, GPS/시간 정보 추출 | P0 |
| **동선 지도 시각화** | 선택한 사진들의 위치를 지도에 순서대로 표시 | P0 |
| **타임라인 생성** | 시간순 정렬, 장소별 그룹핑 | P0 |
| **AI 활동 추론** | 장소 유형 + 시간대 기반 활동 추측 | P0 |
| **결과 카드 생성** | SNS 공유용 이미지/텍스트 생성 | P0 |

#### 🔹 Phase 2: Enhanced Features

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| **여행 vs 일상 자동 분류** | 이동 거리/패턴 기반 자동 판단 | P1 |

| **여행 통계** | 총 이동거리, 방문 장소 수, 주요 지역 | P1 |
| **블로그 내보내기** | Markdown/HTML 포맷 내보내기 (네이버 블로그, 티스토리 등) | P1 |

#### 🔹 Phase 3: Advanced Features

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| **여행 스토리 자동 생성** | AI가 전체 여행을 텍스트로 요약 | P2 |

### 4.2 AI 활동 추론 로직 (예시)

```
┌─────────────────────────────────────────────────────────────────┐
│                    AI 활동 추론 Flow                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│  Input: GPS 좌표 + 촬영 시간                                      │
│                                                                   │
│         ↓                                                         │
│                                                                   │
│  Step 1: Reverse Geocoding                                        │
│  • GPS → 주소, 장소명, 장소 유형(카페, 공원, 관광지 등)              │

│                                                                   │
│  Step 2: 시간대 분석                                               │
│  • 아침(6-11), 점심(11-14), 오후(14-18), 저녁(18-22), 밤(22-6)     │
│                                                                   │
│         ↓                                                         │
│                                                                   │
│  Step 3: 패턴 매칭                                                 │
│  ┌─────────────────────────────────────────────────────────┐     │
│  │ 장소: 카페 + 시간: 오전 10시 → "브런치/카페 타임"          │     │
│  │ 장소: 공원 + 시간: 오후 3시 → "산책/피크닉"               │     │
│  │ 장소: 레스토랑 + 시간: 저녁 7시 → "저녁 식사/모임"        │     │
│  │ 장소: 공연장 + 시간: 저녁 8시 → "공연/콘서트 관람"        │     │
│  │ 장소: 관광지 + 여러 장소 이동 → "여행 중"                │     │
│  └─────────────────────────────────────────────────────────┘     │
│                                                                   │
│         ↓                                                         │
│                                                                   │
│  Output: "오늘의 하루"                                             │
│  "성수동 카페에서 브런치로 시작 → 한강공원에서 산책 →               │

│                                                                   │

│  • 오프라인에서도 완전한 기능 제공                                  │

```

### 6.2 데이터 소스 (On-Device)

| 데이터 | 출처 | 처리 방식 |
|--------|------|-----------|
| GPS 좌표 | 사진 EXIF 메타데이터 | iOS PhotoKit |
| 촬영 시간 | 사진 EXIF 메타데이터 | iOS PhotoKit |
| 장소 정보 | **Apple CLGeocoder** | On-device Reverse Geocoding (iOS 내장) |
| 지도 시각화 | **Apple MapKit** | 네이티브 지도 (외부 API 불필요) |

### 6.3 On-Device 기술 스택

| 기술 | 용도 | 비고 |
|------|------|------|
| **PhotoKit** | 사진 접근 및 메타데이터 추출 | iOS 기본 프레임워크 |
| **Core Location (CLGeocoder)** | GPS → 주소 변환 | Apple 서버 사용하지만 무료, 기기 캐싱 |
| **MapKit** | 지도 표시 및 경로 시각화 | iOS 기본 프레임워크 |
| **Core ML** | 장소 유형 분류 (선택적) | 로컬 ML 모델 |
| **SwiftUI / UIKit** | UI 구현 | - |

✅ 프라이버시 설계 원칙

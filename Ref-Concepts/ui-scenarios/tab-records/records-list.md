### 9.11 Context별 기록 카드 스타일

> **v3.2 업데이트**: Context에 따라 카드 표시 정보 차별화. 기록 상세 화면 진입 시에도 Context별 UI 게이팅 적용:
> - **일상/외출**: Wander Intelligence 섹션 미표시, AI 다듬기 버튼 미표시
> - **여행/혼합**: Wander Intelligence 섹션 표시 (StoryPreviewCard + InsightsPreview만, TravelDNA/TripScore/MomentScore 제거됨), AI 다듬기 버튼 표시
> - 지도 제목: 일상="이동 경로"(미표시), 외출="외출 동선", 여행="여행 동선"

### 9.12 검색 및 필터

**검색 기능:**
- 검색어 입력 시 제목에서 실시간 필터링
- 빈 결과 시: "검색 결과가 없습니다" 표시
- 검색어 지우기 버튼 (X)

**Context 필터 칩:**
| 필터 | 설명 |
|------|------|
| 전체 | 모든 기록 표시 |
| ✈️ 여행 | 여행 Context 기록 |
| 🚶 외출 | 외출 Context 기록 |
| 🏠 일상 | 일상 Context 기록 |
| 📅 주간 | 주간 하이라이트 기록 |

> **v3.1 변경**: 기존 "카테고리" 개념을 "Context"로 통합. 사용자 정의 카테고리 기능은 유지하되, 자동 분류된 Context와 별도로 관리.

### 9.14 Context 변경 시트

> **v3.1 신규**: 자동 분류가 잘못된 경우 사용자가 수동으로 변경

```
┌─────────────────────────────────┐
│                                 │
│  Context 변경                   │
│                                 │
│  현재: 🏠 일상                  │
│                                 │
│  ┌─────────────────────────┐   │
│  │  🏠 일상                 │   │
│  └─────────────────────────┘   │
│  ┌─────────────────────────┐   │
│  │  🚶 외출                 │   │
│  └─────────────────────────┘   │
│  ┌─────────────────────────┐   │
│  │  ✈️ 여행                 │   │
│  └─────────────────────────┘   │
│                                 │
│  💡 변경하면 결과 화면 UI도      │
│     해당 Context에 맞게 변경됩니다 │
│                                 │
│  [취소]                         │
│                                 │
└─────────────────────────────────┘
```

**동작:**
- Context 변경 시 TravelRecord.context 필드 업데이트
- 상세 화면 진입 시 변경된 Context에 맞는 UI 표시
  - 일상/외출로 변경 시: Wander Intelligence 섹션 숨김, AI 다듬기 버튼 숨김, 지도 제목 변경
  - 여행/혼합으로 변경 시: Wander Intelligence 섹션 표시 (데이터가 있는 경우), AI 다듬기 버튼 표시
- 학습 데이터에는 영향 없음 (사용자 수동 변경이므로)
- **주의**: 일상/외출에서 여행으로 변경해도 Wander Intelligence 데이터가 없으면 해당 섹션은 비어있음 (분석 시점에 스킵되었으므로)
